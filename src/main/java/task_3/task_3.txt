Расскажите, как в выбранном вами языке программирования поддерживается концепция "класс как модуль".

Помимо того, что можно переиспользовать функциональность класса, если наследоваться от него с помощью слова extends, есть еще следующие способы переиспользовать код:
1. Можно организовать код в библиотеку. Библиотека - набор классов с некой функциональностью, организованных в пакеты.
Пакеты позволяют организовать уникальное пространство имен для своего содержимого, где каждое название класса уникально внутри пространства.
Это ведет к тому, то в разных библиотеках могут быть классы с одинаковым названием, но конфликтов при этом не будет.
Чтобы подключить библиотеку, нужно поместить скомпилированный jar-файл библиотеки в classpath - это переменная окружения (список директорий, jar-файлов) где JVM при запуске ищет классы для запуска.
3. Есть системы сборки - maven, gradle. Maven позволяет организовать проект в виде модулей. Каждый модуль может быть скомпилирован отдельно в jar\war файл и использован в другом модуле/проекте.
4. В Java 9 появились модули, которые также могут быть скомпилированы отдельно, аналогично модулю в maven.
Вместе с этим, например, появилась возможность создавать кастомизированные JRE, которые включают лишь необходимые модули.
Благодаря этому можно исключить ненужные зависимости и уменьшить итоговый вес программы.
