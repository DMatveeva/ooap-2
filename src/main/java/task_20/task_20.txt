1. Наследование вариаций.

1.1 Наследование с функциональной вариацией.
Пусть имеется класс Dancer. И есть потомки BalletDancer, HipHopDancer, BreakDancer.
Если они отличаются лишь тем, что переопределяют родительский метод dance(), то такое наследование можно назвать наследованием с функциональной вариацией.

1.2 Наследование с вариацией типа.
Имеется класс Курьер. Допустим, он имеет единственный метод deliver(Package package).
От него наследуется класс "Курьер документов". Он только переопределяет метод deliver(Document document).
В Java при изменении сигнатуры метода, считается, что он не переопределяет метод с таким же именем в родительском классе.
Компилятор выдаст ошибку, если на такой метод поставить аннотацию Override.

2. Наследование с конкретизацией.
Наверное тут можно привести в пример класс Object в Java. ОВсе остальные объекты наследуются от него, и представляют собой конкретные реализации.

3. Структурное наследование.
Имеется класс "Водоплавающее". В качестве подтипов могут выступать "Морж", "Пингвин".
